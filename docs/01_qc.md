# 01 — Data intake and QC (FastQC / MultiQC)

## Purpose
This step performs basic sequencing QC on raw demultiplexed FASTQ files (R1/R2) for STARR-seq RNA and the corresponding input DNA (and any other FASTQ sets processed by the pipeline). The goal is to detect obvious sequencing and library-prep issues early and to generate a single MultiQC report for sharing.

## Inputs
- Demultiplexed FASTQ files (compressed `.fastq.gz`), typically paired-end:
  - `*_R1*.fastq.gz`
  - `*_R2*.fastq.gz`

Expected to include (depending on experiment):
- STARR-seq RNA (output) FASTQs
- Input DNA FASTQs
- Template-switch / TSS libraries (if included in the same analysis batch)

## Outputs
- Per-FASTQ FastQC reports:
  - `fastqc_results/*_fastqc.html`
  - `fastqc_results/*_fastqc.zip`
- Aggregated MultiQC report:
  - `multiqc_results/multiqc_report.html`
  - `multiqc_results/multiqc_data/`

## Tools
- FastQC
- MultiQC

## What we check (key FastQC modules)
FastQC provides several standard checks. For STARR-seq, the most informative ones are:

1. **Per base sequence quality**
   - Expect high quality across cycles, but some drop near read end is common.
2. **Per sequence quality scores**
   - Large low-quality tail suggests issues with sequencing run or aggressive read-through/adapters.
3. **Per base sequence content / base composition**
   - STARR-seq often has non-random composition due to primers/adapters and read structure.
   - Strong biases at the start can be expected if constant sequence is present.
4. **Per base GC content / per sequence GC content**
   - Large deviations may indicate contamination or unexpected library composition.
5. **Adapter content**
   - Presence suggests adapter read-through and may require trimming.
6. **Overrepresented sequences**
   - Can indicate primer dimers, adapter contamination, PhiX, rRNA, or other repeated artefacts.
7. **Sequence duplication levels**
   - For ultra-complex STARR-seq, duplication can still be high when sequencing depth oversamples the library.
   - Duplication should be interpreted together with complexity estimates (later steps).
8. **K-mer content**
   - Enrichment of k-mers can indicate adapter/primer sequence or technical artefacts.
